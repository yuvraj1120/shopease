🔄 Daily Workflow — 9 Steps (Do This Every Day)
1
🌅 Start on main — get latest
Always begin by switching to main and pulling the latest from GitHub. Never start on old code.
# Switch + get latest
git checkout main
git pull origin main
2
🌿 Create feature branch
Never work on main. Every change gets its own branch. Name it clearly.
# Create + switch in 1 command
git checkout -b feature/name
* feature/name ← you are here
↓    work on your files in VS Code    ↓
3
📋 Stage your changes
git add moves files into the staging area — like putting items in a box before sealing it.
git add .
# or specific file:
git add src/ProductController.java
# verify:
git status
4
💾 Commit — save snapshot
git commit seals the box and labels it. Saved to your local repo forever.
git commit -m "feat(docker): add Dockerfile"
[feature/.. 4f2c8a1] feat(docker)...
1 file changed, 18 insertions(+)
5
☁️ Push branch to GitHub
Uploads your commits from local repo to GitHub remote. First push needs origin + branch name.
git push origin feature/name
* [new branch] feature/name → feature/name
6
🔀 Open Pull Request on GitHub
Go to github.com/yuvraj1120/shopease. A yellow banner appears — click it.
🟡 "Compare & pull request"
→ Fill Title + Description
→ base: main ← compare: feature/
→ Click: "Create pull request"
7
✅ Merge the PR
After review, merge. Your feature code is now part of main. Badge turns purple.
→ Click: "Merge pull request"
→ Click: "Confirm merge"
🟣 Successfully merged!
→ Click: "Delete branch"
8
🧹 Clean up + pull main
Delete branch locally and pull merged code. Your local main now matches GitHub exactly.
git checkout main
git pull origin main
git branch -d feature/name
🟩 Green square earned! ✓

🔀 3 Merge Strategies — Know the Difference (Interview Topic)
🔀 Merge Commit ✅ Use this
Combines branches + creates a merge commit M. Full history preserved. You can see every feature commit.
main: C1─C2──────────M \ / feat: F1─F2─F3
🗜️ Squash Merge Use for cleanup
All feature commits (F1+F2+F3) become ONE clean commit on main. Good for messy "fix", "oops" commits.
main: C1─C2─[F1+F2+F3] ↑ 1 clean commit no branch visible
♻️ Rebase ❌ Never on shared
Replays commits on top of main. Linear history. NEVER on shared branches — rewrites hashes.
main: C1─C2─F1'─F2'─F3' ↑ linear, no merge commit rewrites commit history

🚨 Undo Mistakes — Stay Calm and Fix It
😬 Wrong commit message — not pushed yet
git commit --amend -m "correct message" # Changes the last commit message # Only works BEFORE pushing!
😬 Committed wrong files — not pushed yet
git reset --soft HEAD~1 # Uncommits but keeps files staged git add correct-file.java git commit -m "correct message"
😱 Already PUSHED to shared branch
# SAFE way — adds an undo commit git revert <commit-hash> git push origin main # ✅ History preserved, safe for team # ❌ Never use git reset on pushed commits
⚡ Mid-work, need to switch branch urgently
git stash # shelve your work git checkout main # switch branch # ... fix the urgent thing ... git checkout feature/your-branch git stash pop # restore your work
😱 Accidentally committed a .pem AWS key!
# EMERGENCY — do this immediately: # 1. Go to AWS → IAM → delete the key NOW # 2. Remove file + add to .gitignore # 3. git rm --cached filename.pem # 4. git commit + git push --force # ⚠ Prevention: .gitignore set up on Day 2
🔍 Deleted a branch by mistake
# Branches can be recovered! git reflog # Find the last commit hash of that branch git checkout -b recovered-branch <hash> # Git keeps reflog for ~30 days
